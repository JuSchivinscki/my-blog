# Usando IA para criar projetos que realmente fazem sentido

## Abertura

Tentei algumas vezes usar a IA para me ajudar a criar projetos e ela ficava alucinando.

Foi quando eu percebi que o problema era eu, e não ela.

Eu só percebi esse problema quando fui ler o artigo **"AI in Product Development: Netflix, BMW, and PepsiCo"**, que fala sobre as dificuldades de criar produtos e como essas grandes empresas conseguiram contornar seus maiores desafios usando IA.

A partir disso, percebi que, assim como elas, também podemos treinar a IA para contornar os nossos desafios. A diferença é que eu não preciso criar um produto revolucionário; eu só preciso treiná-la para me ajudar a praticar o que estou aprendendo.

Foi aí que surgiu a ideia de usar a IA para me ajudar a criar projetos.


## Contexto & Tensão

Existe um ciclo que praticamente todo desenvolvedor já viveu.

O loop é:

> Estudar a teoria → tentar praticar → travar sem ideia de projeto → voltar para a teoria.

E assim a ideia do projeto extraordinário nunca chega.



## Desenvolvimento

A ideia perfeita não tinha chegado naquele dia.

Conversando com meu pai, ele comentou um problema que percebe com frequência em clínicas médicas: agendas ociosas.

Durante essa conversa fizemos um verdadeiro _role play_. Fui fazendo anotações, perguntando detalhes e validando tudo com ele.

Só que chegou um momento em que cada nova ideia deixava o sistema mais complexo.

Foi aí que precisei ser firme e dizer:

> "Olha, acho que para a primeira versão essa ideia não se encaixa. Ela aumenta muito a complexidade do desenvolvimento e vai atrasar toda a entrega."

Difícil, né?

Depois resolvi pedir ajuda para a IA.

E ela fez exatamente a mesma coisa.

Começou a criar user stories complexas, relações de banco de dados, regras de negócio, backlog gigantesco...

O projeto ficou ainda maior.

A diferença é que meu pai não era um cliente de verdade.

Ele não se importou muito quando o projeto acabou não indo para frente.

Mas...

**E se fosse um cliente real?**

## Virada ou Insight

Meu problema não era usar IA para planejar.

Era usar IA em um projeto que já estava grande demais, sem antes treiná-la para entender meu contexto.

Ela alucinava porque não conhecia:

- meus objetivos;
- meu nível técnico;
- minhas dificuldades;
- minhas limitações.

Quando apresentei um projeto enorme, ela simplesmente assumiu que eu sabia exatamente o que estava fazendo.

E foi justamente aí que o contexto errado fez nós duas alucinarmos juntas.

A solução foi criar a **Strategic Learning Architect**, uma _skill_ responsável por transformar teoria em prática.

Dentro dela existe uma sub-skill chamada **project-simulation-system**, cuja função é simular clientes, produtos e problemas reais para gerar projetos compatíveis com aquilo que estou aprendendo.

Treinar, nesse caso, significa fornecer um conjunto de instruções permanentes sobre mim:

- meus objetivos;
- meu nível;
- minhas dificuldades;
- minha forma de aprender.

No Claude, por exemplo, isso pode ser feito usando memória persistente.

Mas o comportamento mais importante que ensinei foi:

> Nunca assumir. Sempre perguntar e confirmar antes de executar.


## Prompt para praticar uma tecnologia específica

```
Ative o project-simulation-system.

Estou estudando React.

Assuma o papel de um cliente real.

Não me dê exercícios.

Crie um problema de negócio realista.

Inclua:

- contexto da empresa;
- problema;
- objetivos;
- requisitos;
- funcionalidades;
- restrições;
- critérios de sucesso.

Não me entregue a solução.

Aja como um cliente real contratando uma desenvolvedora.
```

---

## Prompt para começar pelo problema

```
Ative o project-simulation-system.

Não quero começar pela tecnologia.

Quero começar pelo problema.

Me ajude a encontrar problemas reais que poderiam ser resolvidos com software.

Considere:

- estudantes;
- desenvolvedores;
- aprendizado;
- idiomas;
- produtividade;
- carreira.

Depois transforme o melhor problema em uma oportunidade de projeto.
```

## O teste

Para mostrar a diferença na prática, fiz o teste mais simples possível.

Sem ativar a sub-skill.

Sem fornecer contexto.

Usei apenas:

Estou estudando React.

Crie um projeto para eu praticar.


Mesmo assim, como ela já carregava meu contexto, a resposta foi completamente diferente.

Em vez de entregar um projeto pronto, ela começou perguntando e confirmando informações antes de continuar.

---

Antes eu passava horas tentando pensar em um projeto para praticar.

Hoje eu simplesmente começo a desenvolver.

Antes de pedir uma ideia para uma IA, você precisa dar um bom contexto.

Se quiser testar isso hoje, experimente incluir no prompt:

- seu nível atual;
- quanto tempo você tem disponível;
- exatamente o que deseja aprender.

Quanto mais específico, melhor.

No fim, o mais importante é que a IA saiba duas coisas:

- onde você está;
- para onde você quer chegar.
